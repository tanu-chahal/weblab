🔐 OTP Inputs Test

Six 6-digit OTP input styles for verifying TOTP field detection and autofill. Each specimen lists the OTP-related attributes it carries; the last one carries none (control).

1. Simple single input

A plain text field that announces itself as an OTP field via attributes and label.

name="otp" · id="otp" · autocomplete="one-time-code" · inputmode="numeric" · pattern="[0-9]{6}" · maxlength="6"

Value: —

2. Six separate boxes (classic)

The pattern most sites use: six single-character boxes with auto-advance, backspace navigation, and paste-to-distribute.

six inputs name="otp-1..6" · maxlength="1" · inputmode="numeric" · autocomplete="one-time-code" on first box

Value: —

3. Six blank spaces (underline style)

Borderless cells with only an underline — same six-input behavior, a different visual treatment.

six inputs name="code-1..6" · maxlength="1" · inputmode="numeric" · autocomplete="one-time-code" on first box

Value: —

4. Segmented single field

Looks like six cells but is a single input — the segmentation is purely a CSS letter-spacing illusion. Tests detectors that assume multi-box = multiple inputs.

name="one_time_code" · autocomplete="one-time-code" · inputmode="numeric" · maxlength="6"

Value: —

5. Authenticator code (alternate naming)

A single field that uses different OTP-ish naming and a placeholder, to exercise detectors that match on broader keyword sets.

name="authenticatorCode" · id="verification-code" · autocomplete="one-time-code" · inputmode="numeric" · placeholder="123 456"

Value: —

6. Control field (no OTP hints)

A plain text input with no OTP-related attributes. Your flow should not detect this as an OTP field.

name="fullName" · id="full-name" · autocomplete="name" · type="text" — no OTP signals
